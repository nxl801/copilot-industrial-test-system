# Degradation Strategy

## Keep in VS Code interactive sessions
- bench dry-run
- real bench action approval
- on-site abnormal log interpretation
- tasks requiring extension tools or authenticated enterprise systems

## Move to CLI
- batch read-only analysis
- independent parallel work
- long-running background organization tasks that do not touch hardware

## Must be MCP-wrapped
- bench lock / reserve / release
- power cycle / download / capture start-stop
- scan import / normalization
- test management / artifact attach / defect stub
- PLC simulation / motion trace / parameter compare

## GitHub.com coding agent downgrade
- remove `argument-hint`
- remove `handoffs`
- do not rely on local `.vscode/mcp.json`
- keep only read-only / report / docs / code synthesis capabilities
