# Public Release Checklist

Before publishing to GitHub as a public repository:

1. Remove any real credentials, internal URLs, customer names, and proprietary build IDs
2. Keep only demo artifacts and synthetic example data
3. Ensure MCP servers are stub/demo-safe by default
4. Add clear warnings that bench actions are disabled by default and require explicit human approval
5. Provide quickstart docs, profile matrix, and demo instructions
6. Include LICENSE, README, CONTRIBUTING, SECURITY, and CHANGELOG files
7. Verify no private screenshots, logs, scan bundles, or topology files remain
